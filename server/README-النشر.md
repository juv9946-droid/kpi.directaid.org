# نشر نظام مؤشرات الأداء على الإنترنت (خطوات بدون خبرة برمجية)

النظام الآن مكوّن من جزأين في مجلد المشروع:
- **الواجهة الأمامية**: ملف `نظام KPI - تصميم Hi-Fi.dc.html` مع `support.js` و`cloud-sync.js`.
- **الخادم**: مجلد `server/` (Node.js + Express + MySQL).

الخادم يخدم الواجهة ويحفظ البيانات في قاعدة MySQL مركزية، فيصبح لديك **رابط واحد** يستخدمه كل الموظفين في نفس الوقت.

---

## الخيار الأول: Railway (موصى به — يوفّر MySQL جاهزًا)

### 1) ارفع المشروع إلى GitHub
- أنشئ حسابًا على github.com ← زر **New repository** ← أعطه اسمًا ← **Create**.
- اضغط **uploading an existing file** وارفع **كامل مجلد المشروع** (الملف HTML + `support.js` + `cloud-sync.js` + مجلد `server`).

### 2) أنشئ مشروعًا على Railway
- ادخل railway.app وسجّل الدخول بحساب GitHub.
- **New Project** ← **Deploy from GitHub repo** ← اختر المستودع الذي رفعته.

### 3) أضِف قاعدة بيانات MySQL
- داخل المشروع: **New** ← **Database** ← **Add MySQL**.
- سيُنشئ Railway متغيّرًا باسم `MYSQL_URL` تلقائيًا.

### 4) اضبط إعدادات الخدمة
- افتح خدمة التطبيق (وليس قاعدة البيانات) ← **Settings**:
  - **Root Directory**: اكتب `server`
  - **Start Command**: اكتب `npm start`
- في تبويب **Variables** تأكّد من وجود `MYSQL_URL` (اربطه من قاعدة MySQL إن لزم عبر «Variable Reference»).

### 5) شغّل وانسخ الرابط
- Railway يبني ويشغّل تلقائيًا. من **Settings ← Networking ← Generate Domain** تحصل على رابط عام مثل:
  `https://kpi-system.up.railway.app`
- هذا هو الرابط الذي تشاركه مع الموظفين. يعمل من الكمبيوتر والجوال عبر المتصفح فقط.

---

## الخيار الثاني: Render

Render لا يوفّر MySQL مباشرة، لذا نستخدم قاعدة MySQL خارجية مجانية (Railway أو Aiven أو PlanetScale) ونصل بها.

### 1) جهّز قاعدة MySQL
- أنشئ قاعدة MySQL على Railway (كما أعلاه) أو Aiven، وانسخ رابط الاتصال `mysql://...`.

### 2) أنشئ خدمة على Render
- ادخل render.com ← **New** ← **Web Service** ← اربط مستودع GitHub.
- **Root Directory**: `server`
- **Build Command**: `npm install`
- **Start Command**: `npm start`

### 3) أضِف متغيّر البيئة
- في **Environment** أضِف متغيّرًا:
  - المفتاح: `MYSQL_URL`  ← القيمة: رابط الاتصال الذي نسخته.

### 4) انشر
- **Create Web Service**. بعد اكتمال البناء يعطيك Render رابطًا عامًا — هذا رابط النظام.

## ملاحظات مهمة
- **حساب واحد مشترك**: لا يوجد نظام تسجيل دخول؛ أي شخص لديه الرابط يمكنه الإضافة والتعديل — كما طلبت.
- **التحديث المباشر**: تظهر تعديلات أي موظف لدى الباقين خلال ثوانٍ (يتم سحب التغييرات كل ٤ ثوانٍ).
- **النسخ الاحتياطي**: قاعدة MySQL على المنصة تحتفظ بكل البيانات؛ يُنصح بتفعيل النسخ الاحتياطي من لوحة المنصة.
- **التشغيل محليًا للتجربة** (اختياري): داخل مجلد `server` نفّذ `npm install` ثم انسخ `.env.example` إلى `.env` واملأ بيانات MySQL، ثم `npm start`، وافتح `http://localhost:3000`.

## هل تريد حماية بسيطة بكلمة مرور واحدة؟
أخبرني وأضيف شاشة دخول واحدة (كلمة مرور مشتركة) قبل الوصول للنظام — دون أدوار أو صلاحيات.
